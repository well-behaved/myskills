# 线上 SQL 操作手册归档

将重要的线上数据库操作记录为飞书文档，形成可复用、可追溯的操作流水手册。

---

## 执行方式

**必须使用 Agent tool 派发子 agent 执行**，避免污染当前对话上下文。

---

## 输入解析

从用户输入或当前对话上下文中提取以下信息：

| 字段 | 来源 | 必须 |
|------|------|------|
| 需求主题 | 用户说明或从 SQL 语义推断 | 是 |
| 需求背景 | 用户描述的业务原因 | 是 |
| 执行 SQL | 实际要在线上跑的 SQL | 是 |
| 回滚 SQL | 用于撤销变更的 SQL | 否（尽量生成） |
| 目标目录 | 飞书 wiki_node | 否 |

**如果信息不完整**，向用户询问缺失字段后再派发子 agent。

---

## 默认配置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| wiki_node | `RosewlDjyi7pz9ko0XjcayF4npb` | 飞书归档目录，用户可通过参数覆盖 |
| 文档标题格式 | `{需求主题} {YYYY-MM-DD}` | 自动拼接当前日期 |

---

## 子 Agent 派发指令

派发子 agent 时，prompt 必须包含以下完整信息：

```
你需要在飞书创建一份线上变更记录文档。

**文档标题**: {需求主题} {YYYY-MM-DD}
**目标目录 wiki_node**: {wiki_node}

**文档内容（Markdown）**:

## 需求背景

{需求背景描述}

---

## 线上执行 SQL

```sql
{执行SQL}
```

---

## 回滚 SQL

```sql
{回滚SQL}
```

请调用 mcp__feishu-mcp__create-doc 工具，参数：
- title: 文档标题
- wiki_node: 目标目录
- markdown: 上述文档内容

完成后返回文档链接。
```

---

## 文档模板

生成的文档固定包含以下结构：

1. **需求背景** — 为什么要做这次变更，筛选条件，影响范围
2. **线上执行 SQL** — 按步骤组织（预览 → 执行 → 验证）
3. **回滚 SQL** — 出问题时的恢复手段

---

## 注意事项

- 回滚 SQL 如果用户未提供，根据执行 SQL 语义自动生成（INSERT → DELETE，UPDATE → 反向 UPDATE）
- 日期取当天日期
- 如果用户提供了飞书目录链接（如 `https://mi.feishu.cn/wiki/xxx`），从中解析 wiki_node
- 文档正文开头不要重复标题（飞书会自动显示标题）
